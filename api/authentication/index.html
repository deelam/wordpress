Authentication

Authenticate by sending a POST request to the /auth/realms/{customerId}/protocol/openid-connect/token endpoint. Retrieve the access_token from the response body, and use it in the Authorization field in the header for all subsequent requests.

Example request

curl -X POST \
  https://app.alegion.com/auth/realms/{customerId}/protocol/openid-connect/token \
  -H 'cache-control: no-cache' \
  -H 'content-type: application/x-www-form-urlencoded' \
  -d 'client_id=platform-web&username={username}&password={password}&grant_type=password'

Note that the username and password will need to be URL encoded. For example, first+last@company.com would be encoded into first%2Blast%40company.com.

Example response

{
    "access_token": "your_access_token",
    "expires_in": 3600,
    "refresh_expires_in": 1800,
    "refresh_token": "example_refresh_token",
    "token_type": "bearer",
    "not-before-policy": 0,
    "session_state": "bdfa26c7-cd3f-4709-b77d-8e05320454a8"
}

The expires_in field specifies (in seconds) when the access_token will no longer be valid. You will need to reauthenticate to acquire a new access_token.

Contact an Alegion representative to get your customer ID. This is synonymous with tenantId (in the API documentation) and realm (in the https://app.alegion.com/auth/realms URL).

Access tokens have a defined lifespan, and requests made using an expired token will fail with an HTTP 401 error. Simply make the authentication request again and use the new access token in all subsequent requests.
