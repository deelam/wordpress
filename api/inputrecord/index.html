InputRecord

Once a Batch has been created, InputRecords can be added to that Batch. There are several ways to do this.

The import endpoints currently limit imports to 1,000 records per request. Attempting to send more than 1,000 records in a single request will result in an HTTP 422 Unprocessable Entity error. To solve this, split your InputRecords into 1,000 chunks and make several import requests using the same batchId.

The metadata field, which is optional when sent in a request body or as a CSV column, will be associated with that InputRecord throughout the Workflow. Using metadata is a great way to attach additional information to an input record, which might otherwise be difficult to identify in reporting or search scenarios. Note metadata is a reserved word and must be a sibling to the input record data field, not a child of the data field.

Add a Single InputRecord

Load a single record into a batch by issuing a POST request to the /api/v1/batches/{batchId}/records endpoint with a NewInputRecord as the request body.

Example request

curl -X POST \
  https://app.alegion.com/api/v1/batches/{batchId}/records \
  -H 'authorization: Bearer {access_token}' \
  -H 'content-type: application/json' \
  -d '{
	"data": {
        "sample_key": "sample_value"
	},
	"metadata": "my record's metadata"
}'

Example response

{
    "metadata": "my record's metadata",
    "data": {
        "sample_key": "sample_value"
    },
    "createdAt": "2017-11-15T17:55:44.113Z",

    "id": "046b6c7f-0b8a-43b9-b35d-6489e6daee91",
    "status": "queued"

Add Records Using a CSV file

To load input records that are specified in a CSV file, issue a POST call to the /api/v1/batches/{batchId}/records/import-csv endpoint with a UTF-8 encoded CSV file as form data. This is a synchronous operation.

Example Request

curl -X POST \
  https://app.alegion.com/api/v1/batches/{batchId}/records/import-csv \
  -H 'authorization: Bearer {access_token}' \
  -H 'cache-control: no-cache' \
  -H 'Content-Type: multipart/form-data' \
  -F file=@/path/to/records.csv

Example Response

[
    {
        "id": "309ec5c5-6589-47fc-bd3d-c5b4edeb5d57",
        "data": {

        },
        "metadata": "sample metadata",
        "createdAt": "2017-11-15T14:26:17.661Z",
        "workflowStageId": "string",
        "status": "queued"
    }
]

Add Records Using a JSON file

To load records into a batch using a JSON file, issue a POST to the /api/v1/batches/{batchId}/records/import endpoint with a JSON array of InputRecords as the request body.

Example request

curl -X POST \
  https://app.alegion.com/api/v1/batches/{batchId}/records/import \
  -H 'authorization: Bearer {access_token}' \
  -H 'content-type: application/json' \
  -d '[
    {
        "data": {
            "sample_key": "sample_value"
        },
        "metadata": "sample_metadata"
    },
    {
        "data": {
            "sample_key": "another_sample_value"
        },
        "metadata": "more_sample_metadata"
    }
]'

Example response

[
    {
        "id": "309ec5c5-6589-47fc-bd3d-c5b4edeb5d57",
        "data": {
            "sample_key": "sample_value"
        },
        "metadata": "sample_metadata"
        "createdAt": "2017-11-15T14:26:17.661Z",
        "workflowStageId": "string",
        "status": "queued"
    },
    {
        "id": "83224129-b65f-4533-8f06-d25333a21759",
        "data": {
            "sample_key": "another_sample_value"
        },
        "metadata": "more_sample_metadata"

Retrieve InputRecords

To retrieve InputRecords within a batch, issue a GET request to the /api/v1/batches/{batchId}/records endpoint . Results will be paginated.

Example request

curl -X GET \
  https://app.alegion.com/api/v1/batches/{batchId}/records \
  -H 'authorization: Bearer {access_token}'

Example response

[
    {
        "id": "046b6c7f-0b8a-43b9-b35d-6489e6daee91",
        "createdAt": "2017-11-14T21:02:36.491Z",
        "metadata": "aeiou",
        "data": "aeiou",
        "workflowStageId": "046b6c7f-0b8a-43b9-b35d-6489e6daee91",
        "status": "in-progress"
    }
]
