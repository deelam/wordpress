InputRecord

Once a Batch has been created, InputRecords can be added to that Batch. There are several ways to do this.

The import endpoints currently limit imports to 1k records per request for CSV bulk imports and 10k records per request for JSON bulk imports. Attempting to send more in a single request will result in an HTTP 422 Unprocessable Entity error. To solve this, split your InputRecords into smaller chunks and make several import requests using the same batchId.

Add a Single InputRecord

Load a single record into a batch by issuing a POST request to the /api/v1/batches/{batchId}/records endpoint with a NewInputRecord as the request body.

Example request

curl -X POST \
  https://app.alegion.com/api/v1/batches/{batchId}/records \
  -H 'authorization: Bearer {access_token}' \
  -H 'content-type: application/json' \
  -d '{
	"data": {
          "image_url": "http://fileserver.net/photo12.png",
          "image_timestamp": 1563493639
          "image_guid": "CQ4JAQgOCQQDAQgHBAwDDQ"
	},
	"metadata": "imageset-002"
}'

The metadata field (which is optional when sent in a request body or as a CSV column) will be associated with that InputRecord throughout the Workflow. Internally, the metadata field is stored as an uninterpreted string in its own column in our database, which makes querying faster. A group of InputRecords can have the same metadata value, or each InputRecord can have its own unique metadata value. Using this metadata field is a great way to attach an identifier to a single InputRecords or sets of InputRecords, which might otherwise be difficult to identify in reporting or search scenarios. Note that metadata is a reserved word and must be a sibling to the InputRecord’s data field, not a child of the data field.

The value of the data field is a JSON object with key-value pairs. The names of the keys correspond to field names used in customer-specific Task Templates. The set of key names for your specific workflow will be provided by a Customer Success representative. It is critical that the key names match the field names! Any key name that does not match a field name will be ignored and copied to the ResultRecord. In the example above, the image_guid field is used to uniquely identify the image.

Example response

{
    "metadata": "imageset-002",
    "data": {
          "image_url": "http://fileserver.net/photo12.png",
          "image_timestamp": 1563493639
          "image_guid": "CQ4JAQgOCQQDAQgHBAwDDQ"
    },
    "createdAt": "2017-11-15T17:55:44.113Z",
    "workflowStageId": "046b6c7f-0b8a-43b9-b35d-6489e6daee91",
    "id": "046b6c7f-0b8a-43b9-b35d-6489e6daee91",
    "status": "queued"
}

Note that the Alegion Platform assigns an id to each record for its own internal tracking.

All the key-value pairs specified in the data field of the InputRecord will be copied to the ResultRecord. In addition, output key-value pairs representing processing results will be included for each record.

Typically, images and video are referenced using a URL. In the above example, the URL for an image is used in the image_url key-value pair. For more examples, check out the Use Cases.

While this is possible, it is recommended to use bulk importing to add many records to a batch in a single API request. This is done by specifying multiple records in a CSV file or as a JSON array as described in the following sections.

Add Records Using a JSON file

To load records into a batch using a JSON file, issue a POST to the /api/v1/batches/{batchId}/records/import endpoint with a JSON array of NewInputRecords as the request body. Each array element is a NewInputRecord object, described in the above section.

Example request

curl -X POST \
  https://app.alegion.com/api/v1/batches/{batchId}/records/import \
  -H 'authorization: Bearer {access_token}' \
  -H 'content-type: application/json' \
  -d '[
    {
        "data": {
          "image_url": "http://fileserver.net/photo12.png",
          "image_timestamp": 1563493639
          "image_guid": "CQ4JAQgOCQQDAQgHBAwDDQ"
        },
        "metadata": "imageset-002"
    },
    {
        "data": {
          "image_url": "http://fileserver.net/photo13.png",
          "image_timestamp": 1563498936
          "image_guid": "BgMGDwMACw8NAwULBAwCDg"
        },
        "metadata": "imageset-002"
    }
]'

Add Records Using a CSV file

To load input records that are specified in a CSV file, issue a POST call to the /api/v1/batches/{batchId}/records/import-csv endpoint with a UTF-8 encoded CSV file as form data. This is a synchronous operation.

The CSV file should have a header on the first line that lists all the column names. The column names correspond to field name used in customer-specific Task Templates. See the above Add a Single InputRecord section for more description of these field names. Any column name that does not match a field name will be ignored and copied to the ResultRecord.

A / character in a column name is used to nest JSON objects. For example, a CSV file with column names camera/width and camera/height would be represented in JSON as { "camera": { "width": 600, "height": 400 } }, where the 600 and 400 values are provided in a data row in the file.

Example Request

curl -X POST \
  https://app.alegion.com/api/v1/batches/{batchId}/records/import-csv \
  -H 'authorization: Bearer {access_token}' \
  -H 'cache-control: no-cache' \
  -H 'Content-Type: multipart/form-data' \
  -F file=@/path/to/records.csv

Example Response

[
    {
        "id": "309ec5c5-6589-47fc-bd3d-c5b4edeb5d57",
        "data": {

        },
        "metadata": "sample metadata",
        "createdAt": "2017-11-15T14:26:17.661Z",
        "workflowStageId": "string",
        "status": "queued"
    }
]

Retrieve InputRecords

To retrieve InputRecords within a batch, issue a GET request to the /api/v1/batches/{batchId}/records endpoint . Results will be paginated.

Example request

curl -X GET \
  https://app.alegion.com/api/v1/batches/{batchId}/records \
  -H 'authorization: Bearer {access_token}'

Example response

[
    {
        "id": "046b6c7f-0b8a-43b9-b35d-6489e6daee91",
        "createdAt": "2017-11-14T21:02:36.491Z",
        "metadata": "aeiou",
        "data": "aeiou",
        "workflowStageId": "046b6c7f-0b8a-43b9-b35d-6489e6daee91",
        "status": "in-progress"
    }
]

The query request parameter is used for partial or exact match on the InputRecord’s id and metadata fields.

If present, the status parameter is a String array used to query for input records in the specified state(s). Possible values are queued, in-progress, escalated, exception, canceled, and complete.

InputRecords are processed through stages in the workflow. Each stage has a workflowStageId. If this parameter is specified, then only InputRecords that are currently in or passed the specified stage will be retrieved.
