AccessDeniedException

Sending a malformed or incorrect access token will return an HTTP 500 error with a response body like:

{
    "timestamp": 1510865911864,
    "status": 500,
    "error": "Internal Server Error",
    "exception": "org.springframework.security.access.AccessDeniedException",
    "message": "Incorrect token provided",
    "path": "/api/v1/batches/{batchId}/results"
}

Unable to Authenticate

An expired access token will return an HTTP 401 error and look like:

{
    "timestamp": 1512567181019,
    "status": 401,
    "error": "Unauthorized",
    "message": "Unable to authenticate using the Authorization header",
    "path": "/api/v1/batches/{batchId}/records"
}

Get filtered results

You may filter results using the following querystring parameters.

  • query: Full text search
  • minCreatedAt: Minimum creation date filter
  • maxCreatedAt: Maximum creation date filter

Example Filtering Request: By Date

You can use the minCreatedAt and maxCreatedAt query parameters to filter search results by date.

curl -X GET \
  https://app.alegion.com/api/v1/batches/{batchId}/results?minCreatedAt=2018-01-01T00:00:00.000Z&maxCreatedAt=2018-01-01T23:59:59.000Z \
  -H 'authorization: Bearer {access_token}'

Example Filtering Request: Full-text Search

You can use the query query parameter to perform a full-text search of records within a batch. This is often used to search metadata to identify a particular record or group of records.

curl -X GET \
  https://app.alegion.com/api/v1/batches/{batchId}/results?query=ABC123 \
  -H 'authorization: Bearer {access_token}'

List InputRecords

Use the /api/v1/batches/{batchId}/records endpoint to retrieve records belonging to a batch. Results will be paginated.

Example Request

curl -X GET \
  https://app.alegion.com/api/v1/batches/{batchId}/records \
  -H 'authorization: Bearer {access_token}'

Example Response

[
    {
        "id": "046b6c7f-0b8a-43b9-b35d-6489e6daee91",
        "createdAt": "2017-11-14T21:02:36.491Z",
        "metadata": "aeiou",
        "data": "aeiou",
        "workflowStageId": "046b6c7f-0b8a-43b9-b35d-6489e6daee91",
        "status": "in-progress"
    }
]

Notes

Metadata

metadata is an optional field which, when sent in a request body or added as a CSV column, will be associated with that input record throughout its lifetime in the Alegion system. Using metadata is a great way to attach context to an input record which might otherwise be difficult to identify in reporting or search scenarios. Note metadata is a reserved word and must be a sibling to the input record data field, not a child of data.

Valid:

[
    {
        "metadata": "aeiou",
        "data": {
            "vowels": "aeiou"
        }
    }
]

Invalid:

[
    {
        "metadata": "aeiou",
        "data": {
            "metadata": "aeiou"
        }
    }
]

Import Limits

The import endpoints currently limit imports to 1,000 records per request. Attempting to send more than 1,000 records in a single request will result in an HTTP 422 Unprocessable Entity error.

Modify Batch Info

Use the /api/v1/batches/{batchId} endpoint to update a batch.

Example Request

curl -X PUT \
  https://app.alegion.com/api/v1/batches/{batchId} \
  -H 'Authorization: Bearer {access_token}' \
  -H 'Content-Type: application/json' \
  -d '{
    "id": "{batchId}",
    "name": "Batch Name",
    "workflowId": "{workflowId}",
    "createdAt": "2019-01-01",
    "isEnabled": true,
    "priority": "normal",
    "isActive": true
}'

All the request parameters below are required:

  • id: batch id to update (cannot be changed).
  • name: name that is used to identify the batch.
  • workflowId: workflow id the batch belongs to (cannot be changed).
  • createdAt: any date in the format YYYY-MM-DD (cannot be changed).
  • isEnabled: determines if the batch’s tasks are queued (true or false).
  • priority: required, but deprecated (default value: "normal")
  • isActive: required, but deprecated (default value: true)

Example Response

{
    "id": "3c9fa471-882b-4d92-82d4-ece202606ba6",

]]> Authentication https://deelam.github.io/wordpress/api/authentication/ Thu, 11 Jul 2019 21:04:54 +0000 http://deelam.github.io/wordpress/:8000/?page_id=15 Authenticate by sending a POST request to the /auth/realms/{customerId}/protocol/openid-connect/token endpoint. Retrieve the access_token from the response body for use in an Authorization header in all subsequent requests.

Example Request

curl -X POST \
  https://app.alegion.com/auth/realms/{customerId}/protocol/openid-connect/token \
  -H 'cache-control: no-cache' \
  -H 'content-type: application/x-www-form-urlencoded' \
  -d 'client_id=platform-web&username={username}&password={password}&grant_type=password'

Example Response

{
    "access_token": "access_token",
    "expires_in": 3600,
    "refresh_expires_in": 1800,
    "refresh_token": "example_refresh_token",
    "token_type": "bearer",
    "not-before-policy": 0,
    "session_state": "bdfa26c7-cd3f-4709-b77d-8e05320454a8"
}

Contact an Alegion representative to get your customer ID.

Access tokens have a defined lifespan, and requests made using an expired token will fail with an HTTP 401 error. Simply make the authentication request again and use the new access token in all subsequent requests.
