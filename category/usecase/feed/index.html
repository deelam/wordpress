The following shows its usage in a Linux environment.

> ./alegion_sdk.py -h
usage: alegion_sdk.py [-h] action [actionparam [actionparam ...]]
<snip>

The script expects an action followed by parameters for that particular action. The possible actions correspond roughly to the available API calls on the Alegion Platform.

The first action that must be done before anything else is cred to set credentials (replace all capital words with your credentials): 

> ./alegion_sdk.py cred MY_REALM 'MY_USERNAME' 'MY_PASSWORD'

This will create a alegion-cred.json file that will be used for authentication for subsequent actions. Use quotes around your username and password in case of odd characters. If your credentials change, run the ‘cred’ action again to override the alegion-cred.json file.
For debugging purposes, the alegion-cred.json file is not encrypted so the password is not hidden. Delete the alegion-cred.json file at the end of your session if desired.
When the access_token is expired, authentication will be automatically performed, and the alegion-auth.json file will be updated with the authentication response from the server and will be used for all actions.

Next, let’s list the workflows that are available to your account:

> ./alegion_sdk.py workflows

Note the workflow id in the output. This WORKFLOW_ID will be needed to create a batch named ‘My First Batch’ within the workflow. Use the createBatch action as follows, replacing WORKFLOW_ID with your actual workflow id:

> ./alegion_sdk.py createBatch WORKFLOW_ID 'My First Batch'

The batch name must be unique within the workflow, so an error will occur if the command is issued again. Let’s examine the batches in the workflow using the batches action:

> ./alegion_sdk.py batches WORKFLOW_ID

In the output, note that each batch has a unique id. This BATCH_ID will be used to add input records to the batch for processing through the workflow. There are several ways to add input records. For testing, let’s add one record (replacing BATCH_ID with your actual batch id):

> ./alegion_sdk.py addRecord BATCH_ID '{
     "data": { "sample_key": "sample_value" },
     "metadata": "record-00001"
}'

To show all records within a batch, use the records action:

> ./alegion_sdk.py records BATCH_ID

Note the status field for each record. Once the status shows complete, a result record is created and associated with the batch. To show those results, use the results action:

> ./alegion_sdk.py results BATCH_ID

Each result record will have a metadata field with the same value that was set on the corresponding input record.
