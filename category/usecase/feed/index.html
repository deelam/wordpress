The following shows its usage in a Linux environment.

Download it from python.org or use your OS’s application manager to install it.

The script should still work but the output won’t be as pretty. To fix that, modify all lines that use the print method by removing the outermost parentheses. For example print("Response: ", response) becomes print "Response: ", response.

Install it by running pip install requests.

> ./alegion_client.py -h
usage: alegion_client.py [-h] action [actionparam [actionparam ...]]
<snip>

The script expects an action followed by parameters for that particular action. The chosen action determines which API request will be issued to the Alegion Platform.

The first action that must be done before anything else is cred to set credentials (replace all capital words with your credentials, and use quotes around your username and password in case of odd characters):

> ./alegion_client.py cred MY_REALM 'MY_USERNAME' 'MY_PASSWORD'

This will create a alegion-cred.json file that will be used for authentication. To ensure the credentials are correct, an initial authentication is performed and the response from the server is written to a alegion-auth.json file, which is used for subsequent actions.

If your credentials change, run the ‘cred’ action again to override the alegion-cred.json file. For debugging purposes, the alegion-cred.json file is not encrypted so the password is not hidden. Delete the alegion-cred.json file at the end of your session if desired.

When the access_token is expired, reauthentication will be automatically performed, and the alegion-auth.json file will be updated.

Next, let’s list the workflows that are available to your account:

> ./alegion_client.py workflows

Note the workflow id in the output. This WORKFLOW_ID will be needed to create a batch (which is simply a container for records) named ‘My First Batch’ within the workflow. To do this, use the createBatch action as follows, replacing WORKFLOW_ID with your actual workflow id:

> ./alegion_client.py createBatch WORKFLOW_ID 'My First Batch'

The batch name must be unique within the workflow, so an error response will be returned if the command is issued again. Try it to see what the error looks like. Now, let’s examine the batches in the workflow using the batches action:

> ./alegion_client.py batches WORKFLOW_ID

In the output, find the batch you created, and notice that each batch has a unique id. This BATCH_ID will be used to add input records to the batch for processing through the workflow. There are several ways to add input records. For testing, let’s add one record (replacing BATCH_ID with your actual batch id):

> ./alegion_client.py addRecord BATCH_ID '{
     "data": { "sample_key": "sample_value" },
     "metadata": "record-00001"
}'

You can add input records to a batch at any time using the method above or in bulk using a CSV file or JSON — see the next section for a Bulk Importing Example. There is no limit to the number of input records in a batch or the number of batches in a workflow.

Once an input record is added to a batch, it will be queued up for processing through the workflow. To show all records within a batch, use the records action:

> ./alegion_client.py records BATCH_ID

Note the status field for each record. Once the status shows complete, a result record is created and associated with the batch. To show those results, use the results action:

> ./alegion_client.py results BATCH_ID

Each result record will have a metadata field with the same value that was set on the corresponding input record.

Bulk Importing Example

To add records based on data from each line in a CSV file, use the addRecordsCsv action:

> ./alegion_client.py addRecordsCsv BATCH_ID MYRECORDS.CSV

In the CSV file, the column names depends on the customer-specific workflow. Specific column names are required, and an error will be return if they are missing. A CSV file could look like this:

myData/title,metadata,url
Dog,animal-0001,https://images.net/fido.png
Cat,animal-0002,https://images.net/whiskers.gif

In this example, note that the slash in myData/title creates a nested JSON object:

  {
    "createdAt": "2019-07-22T21:24:11.907Z",
    "data": {
      "image1": "https://images.net/fido.png",
      "myData": {
        "title": "Dog"
      }
    },
    "id": "fdcf4bd5-9229-40ec-9672-e6de73e2e31c",
    "metadata": "animal-0001",
    "status": "queued"
  }

For more details about bulk importing, see Input records.
